# Module 2: Issue Structuring & Framing

An issue is outputted from module 1 and inputted into module 2.
Module 2 sorts the submissions within an issue
into smaller groups of submissions based on subject matter.
This organizes the key aspects of an issue for input into module 3.

## Inputs

**Issue**  
(from module 1)

## Outputs

**Structured Issue View**  
This is inputted into module 3.

## Functionality

### Submissions Within Issue are Clustered

Submissions are grouped together based on how similar they are to each other.

### Submission Clusters are Labeled with a Theme

A label is generated for each cluster based on the cluster's shared subject matter.
